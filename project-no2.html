Hardware (PCB) Design

The electronic PCB design of this device is created using KiCad software, featuring the ESP32 WROOM 32D microcontroller as the main system controller. The device is equipped with an RTC module to maintain accurate time synchronization, a connector for an SD card module as a data storage medium, and an I2C port used to connect the Gamicos Pressure Sensor Transmitter GPT200 for precise pressure measurement, as well as the INA219 sensor for voltage and current measurement. Additionally, a step-down MP1584 module is included to step down the input voltage to match the system's requirements.

3D Design

This is a 3D design for the inner and outer casing of the device. The casing is designed to withstand prolonged exposure to seawater during wave data collection. The inner casing serves to protect and secure the board and battery in place, while the outer casing provides protection for all components from seawater.

Bluetooth Setup

This is the display of the Serial Bluetooth software during the device setup before operation. The available settings include: configuring the time and date, setting the data collection frequency, specifying the start time and date for data collection after the device is powered on, setting the operational and sleep duration, and naming the CSV file for saving data to the SD card.

Device testing

The testing process of this device is conducted through a simulation using a wave flume, where a glass tank generates waves with a height that can be adjusted by the tester. During this process, the device and sensors are tested for data accuracy, which is then verified with a standard sensor.

Development of a Microcontroller-Based Wave Recording System for Coastal Disaster Mitigation

This device is designed to monitor and record sea wave data in real-time, which is crucial for coastal disaster mitigation. Using a microcontroller as the main controller, the device can measure wave height, speed, and frequency along the coastline. The collected data is automatically stored on an SD card and can be analyzed to predict potential disasters, such as tsunamis or coastal flooding. The system is also equipped with a large-capacity battery to support long-term operation without frequent recharging, ensuring the device remains functional in isolated conditions or far from power sources.

This project is the second version of the previous prototype, which used Arduino Mega. The designed PCB functions as a shield for the modules used. In this version 2 prototype, additional features have been added, such as Bluetooth setup, a high data sampling frequency, and an extended battery life capable of lasting approximately 30 days with a work-and-sleep mechanism. The complete features of this device are:

  • Measurement of sea wave height using a pressure sensor connected to an ESP32
  • 40 Hz data sampling frequency
  • Automatic data storage on an SD card for further analysis
  • Device configuration via Bluetooth for easy setup using a smartphone
  • A battery system capable of supporting up to 30 days of independent operation
  • 3D prototype design to ensure the device remains safe and secure in the marine environment for 30 days

The operation of the device starts when it is powered on, and it will search for a Bluetooth connection to configure settings via Bluetooth, such as setting the time, date, start time, and CSV file name. After configuration, the device will not start measuring until the specified time, allowing researchers to deploy it into the sea. The device will begin measuring once it is in the water, and if multiple devices are used, measurements can be taken simultaneously.

The device reads data at a frequency of 40 Hz (or as set by the researcher), meaning it records 40 data points per second. This will continue until the device enters sleep mode to conserve battery. The battery used has a capacity of 27,000 mAh and a voltage of 8.2V.

The collected data is automatically saved to a 4GB SD card. After retrieval from the sea, the data can be analyzed by the researcher. With a 40 Hz sampling frequency, data errors during measurement are minimized, ensuring higher accuracy.
