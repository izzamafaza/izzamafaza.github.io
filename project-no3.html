Dashboard Monitoring Design

The monitoring dashboard, built on the ThingSpeak platform, is designed to provide informative water availability data. The dashboard displays line graphs to monitor water level trends for each device and a map indicating the geographical location of the monitored tanks. The interface showcases data from multiple installed devices. In the example shown, three devices are monitored, with two utilizing dummy data as they are still in the testing phase.

System Design

The system monitors water availability in real-time using the AJ-SR04M ultrasonic sensor to measure water levels, with data processed by the ESP32 microcontroller to calculate the volume based on tank dimensions. Local data is exchanged using the ESP-NOW protocol, while processed data is sent to the ThingSpeak platform via the SIM800L GSM module for remote monitoring. The system operates independently with solar power, charging a 18650 Li-ion battery to ensure functionality in remote areas. ThingSpeak displays data trends and tank locations for more effective water resource management.

Hardware Design

The electronic system consists of key components including the ESP32-S microcontroller for data processing, the AJ-SR04M ultrasonic sensor for measuring water levels, and the SIM800L GSM module for transmitting data to ThingSpeak. Power is managed by a 20Wp solar panel and a TP5100 charging module, which charges a 7.4V Li-ion battery configured in 2S. Voltage regulation is ensured by AMS1117-3.3V for the ESP32, AMS1117-5.0V for the ultrasonic sensor, and MP1584 for the GSM module. A MOSFET N-Channel switch enables sleep mode to reduce energy consumption. The circuit is housed in a waterproof enclosure with secure mounting for installation.

Installation on the Reservoir

The electronic components are placed inside a waterproof box to protect them from environmental factors. A mounting bracket is included for secure installation on the tank and solar panel. Design of a Remote Water Availability Monitoring System on Water Tanks in Drought Areas

This device is designed to provide a remote, real-time water availability monitoring system for communal tanks in drought-prone areas. By utilizing an ultrasonic sensor, the device accurately measures water levels and transmits the data to a monitoring platform, allowing stakeholders to easily track the water condition in each tank. The real-time information helps clean water providers optimize the scheduling of aid deliveries to areas facing supply shortages.

This device is my final project at Universitas Gadjah Mada and is still in the development stage, particularly in terms of selecting the sensor and monitoring dashboard. In this project, I have only reached the testing phase, where I created a single device with a complete design as outlined in the electronics design above, while the remaining devices function as slaves using the ESP32 Devkit 32D. The features of this device are as follows:

  • Real-Time Water Availability Monitoring
  • Local Communication with ESP-NOW
  • Data Transmission to ThingSpeak Platform
  • Independent Operation with Solar Power
  • Work and Sleep Modes on ESP32
  • Parameter Configuration Interface
  • Portable and Sustainable Design

The operation of this device begins with activation, where the system draws power from a battery connected to a solar panel. Once powered on, the system loads initial parameters such as tank height, tank base area, work duration, and sleep duration. If users need to modify settings, they can enter the parameter configuration mode by pressing the button twice. In this mode, a smartphone can connect via Bluetooth to adjust parameters like tank height or work and sleep durations. After completing the configuration, users exit the mode by holding the button for 3 seconds, and the system automatically switches to monitoring mode.

In monitoring mode, the ultrasonic sensor measures the distance between the sensor and the water surface to calculate the water level in the tank. Data from other tanks is collected via slave devices, which lack GSM modules and therefore transmit data to the master device using the ESP-NOW. The master device, equipped with a GSM module, processes the data and transmits it to the ThingSpeak platform in real time. This data is visualized as graphs and tables on the platform, providing users with an easy and informative way to monitor tank water conditions.

After completing the work cycle, the system enters sleep mode to conserve energy, shutting down most components except the ESP32, which operates in low-power mode. During the day, the solar panel recharges the battery while simultaneously powering the system. This workflow repeats through cycles of work and sleep, ensuring the device can continuously monitor water availability efficiently and sustainably, even in areas without electricity or Wi-Fi.
