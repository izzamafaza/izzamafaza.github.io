Dashboard Monitoring Design

The monitoring dashboard is designed to display five measured parameters: water level (m), temperature (°C), TDS (ppm), pH, and turbidity (NTU). It also includes a visualization feature with graphs to record previous monitoring data, as well as information about the device's installation location. Users can easily switch between devices by selecting the Select Device option located in the top-right corner.

Hardware (PCB) Design

The electronic PCB design for this device was created using KiCad software. The device uses the ESP32 WROOM microcontroller as the main system controller. Additionally, a GPS module is included to determine the device's installation location, while a GSM module is used to transmit measurement data to the dashboard without requiring a Wi-Fi connection. The PCB design is built to support the use of two types of sensors: an ultrasonic sensor and a pressure sensor (submersible water level sensor), each capable of accurately measuring water surface levels.

Development of a Rainwater Injection Tool into Dug Wells to Increase Groundwater Level and Groundwater Quality by Utilizing the Internet of Thing (IoT) System

This device is designed to inject rainwater into a dug well with the aim of increasing the groundwater level and improving groundwater quality. By utilizing the Internet of Things (IoT) system, this device allows for real-time monitoring and control, ensuring the effectiveness of the rainwater injection process and supporting the sustainability of groundwater resources.

The project is still in the development stage, with the current focus on basic functions such as measuring the water level using ultrasonic sensors or submersible water level sensors, synchronization with the monitoring dashboard, and managing battery usage to ensure the device can operate autonomously without needing to be connected to an external power source. This device has the following features:

  • Water Level Measurement.
  • Device Location with GPSk.
  • Backend System Based on AWS EC2
  • Efficient and Secure Communication Protocols.
  • Responsive Monitoring Dashboard
  • Real-Time Monitoring
  • Power Saving Mechanism

The operation of this device is as follows:

This device uses a pressure sensor or ultrasonic sensor connected to the ESP32 to accurately measure the water level. The measurement data is sent to a server using the MQTT protocol. The data is then received by Node-RED running on an AWS EC2 cloud server and displayed in real-time via WebSocket on the monitoring dashboard. Additionally, the data is stored in a CSV file, allowing users to view records of previously sent data.

The device is equipped with Bluetooth functionality, enabling the user to set measurement parameters, delivery address, location via GPS, and configure the device’s working and sleep modes. The Bluetooth feature is activated when the device is first powered on, where it will search for a Bluetooth connection for 10 seconds. If no connection is found, the device will continue its operation based on the previous settings. To save power, the device is also equipped with a deep sleep feature, which can be set by the user to allow the device to operate only when necessary for data collection. In deep sleep mode, the MOSFET is triggered to cut off the power supply to all components except the ESP32, further conserving power.
